采购订单接口更新公告 2017-09-13

2017.09.19号上线接口更新公告，主要改动点有：

         1.suning.selfmarket.purchaseorder.query采购订单查询API

（1）返回字段“订单状态”：原10、20、40状态全部返回10状态；

（2）返回字段：新增采购平台订单号；

（3）入参字段“确认标识”：为必填项；

（4）返回字段“确认标识”：10、20、40都默认为“X”；

2.suning.purchaseorderreport.query获取采购订单报表

（1）返回字段“订单状态”：原10、20、40状态全部返回10状态；

3. suning.delivery.reserve 发货预约

（1）增加预计到货日期的校验逻辑：还需校验填写的预计到货日期是否大于最小的订单行的交货日期；大于则报错：“预计入库日期不能大于最小的行项目订单有效期！”

（2）增加预约提交时间的校验逻辑：预约提交不能超过下午5点半，否则无效；

4. suning.selfmarket.reserve.query 预约查询（最新接口）

文化制品相关接口更新公告 2017-06-02

尊敬的各位服务商、云台商户，您们好！

 

文化制品类目的商品发布功能因为业务需要计划将文化制品的商品详情（编辑推荐、作者简介等信息）由现在的商家共享信息，改为商家私有信息，修改后，商品详情的维护与其他类目（电器等）一样，支持详情模块化，详情内容支持维护图片、文字等。

API接口调整如下：

新接口编码

接口名称

对应旧接口

suning.custom.culitem.add

产品申请（文化制品类）新

suning.custom.book.item.add 产品申请（文化制品类）

suning.custom.culitemcontents.modify

商品内容维护（文化制品类）新

suning.custom.book.itemcontents.add 商品内容维护（文化制品类）

suning.custom.culitemcontentrevise.modify

商品内容修改（文化制品类）新

suning.custom.book.itemcontents.modify 商品内容修改（文化制品类）

suning.custom.culitemdetail.query

获取我的商品库详情（文化制品类）新

suning.custom.book.itemdetail.query 获取我的商品库详情（文化制品类）

即用4个新接口替换了4个老接口，新接口相比老接口主要是增加了商品详情字段。

 

以上新接口6.05晚上发布到生产环境，同时老接口将下线，不能使用，请大家及时联调新接口。

部分交易API接口改动点公告 2017-04-24

l  订单查询API

suning.custom.historyorder.query 批量获取历史订单（三个月前的历史订单）

suning.custom.order.query   批量获取订单（三个月内的订单）

suning.custom.order.get   单笔订单查询

suning.custom.ord.query   根据订单修改时间批量查询订单信息

以上API返回参数，新增字段：

isProsupplierDelivery是否供应商发货标识，1代表是

l  发货API

suning.custom.orderdelivery.add 订单发货

suning.custom.orderselfdist.add 订单发货（商家自配送）

以上API新增API错误码、错误描述“供应商发货的订单无法完成发货操作”

商家调发货接口时，对订单包含的所有订单行项目号做校验，如存在代销订单行，则返回报错，整单都无法发货；需要商家调发货接口时过滤掉订单里面的代销订单行；（过滤isProsupplierDelivery为1的订单行项目号）

l  suning.custom.orderdelivery.modify 发货信息修改

新增API错误码、错误描述“供应商发货的订单无法修改发货信息”

商家调发货信息修改接口时，对订单包含的所有订单行项目号做校验，如存在代销订单行，则返回报错，整单都无法修改；需要商家调发货接口时过滤掉订单里面的代销订单行；（过滤isProsupplierDelivery为1的订单行项目号）
