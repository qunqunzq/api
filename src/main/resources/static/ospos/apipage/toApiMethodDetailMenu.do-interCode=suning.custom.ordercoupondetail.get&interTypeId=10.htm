响应参数
名称 类型 示例值 描述
activityCode String 1112008931112 促销活动ID。创建后返回的促销活动ID
activityName String 0元购 活动名称。0元购
startTime String 2015-06-18 00:00:00 活动开始时间。结束时间 > 开始时间 >= 当前时间+系统处理时间（此处为10分钟）
endTime String 2015-07-18 00:00:00 活动结束时间。活动开始时间≤活动结束时间≤活动开始时间+180天
channelInfo String 1,2 活动渠道。31-PC、32-APP、36-WAP、34-电销
activityProductType String 1 活动商品类型。参与活动商品的类型，，1，代表全店铺商品；2，代表指定商品；3排除某部分商品
couponProductType String 1 券使用商品类型。返券能够买的商品集合，1，代表全店铺商品；2，代表指定商品；3排除某部分商品
returnCouponType String 1 返券总量类型。1,不超过多少元；2，不超过多少张
returnCouponValue String 100 返券总量阀值（元/张）。返券总金额，不得超过 999999999元/返券总张数，不得超过 999999999张
activityLimit String 1 活动限制类型。1，限制；2，不限制，当优惠方式为2，订单返券，必填
activityTimesLimit String 10 用户限制总次数。限制用户参加活动的总次数。当活动类型为1时，总次数和每天次数必填其一
peopleActivityTimesLimit String 2 用户每天限制的次数。限制用户每天参加活动的次数。当活动类型为1时，总次数和每天次数必填其一
effectDays String 30 券有效天数。只能到天，最多30天，当优惠方式为2，订单返券，必填
baseQuantifierType String 1 优惠维度。1，订单金额 2，件数
rewardType String 1 优惠方式。当优惠维度为1时，优惠方式可以为1，订单直降；2，订单返券；3、订单包邮；当优惠维度为2时，优惠方式可以为，3、订单包邮；4、订单打折
areaCode String 0012 不包邮区域。省份代码，可以从城市代码获取接口里面获取，如有多个英文逗号隔开。
booleanCap String 1 是否上不封顶。当优惠方式为1，订单直降时，且优惠层级为1时，必填。1代表是，2代表否。
floor String 1 层级。最多5个层级，优惠方式为3时可以不填
discountValue String 100 返券用券条件。单笔订单满多少元才能使用返的券，必须大于返券金额 当优惠方式为2，订单返券，必填
rewardAmount String 5 返券面额。返券的金额，高层级返券的金额必须比低层级要高，最高层级的返券金额不得大于返券总额，当优惠方式为2，订单返券，必填
baseAmount String 100 阀值。满多少件，或者满多少钱，产生优惠，高层级的优惠条件阀值要高于低层级的优惠条件阀值
discountThreshold String 2 打折阀值。打多少折，优惠方式折扣阀值只可输入0.01~9.99，小数点精确两位，当优惠方式为4、订单打折时，必填。高层级的打折阀值要比底层及的打折阀值要高
cutAmount String 20 直降金额。到达优惠阀值后，直降的金额，当优惠方式为1，订单直降时，必填。
couponProductCode String 100021342 苏宁商品编码（券使用商品）。苏宁商品编码，只能填写苏宁通码或者单一商品的商品编码，最多添加200个商品。当券使用商品类型=2或者3时必填。
activityProductCode String 100021342 苏宁商品编码（活动商品）。苏宁商品编码，只能填写苏宁通码或者单一商品的商品编码，最多添加200个商品。当活动商品类型=2或者3时必填。
